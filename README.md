End-to-End Data Pipeline Project Plan (Banking & E-commerce)

This project will build a complete data pipeline and analysis workflow using open data, focusing on banking and e-commerce trends. We will identify measurable patterns (e.g. customer behavior, fraud/anomalies, product popularity), obtain open-source datasets, and implement an ELT process with fully open-source tools. The pipeline will be designed for a 64-bit Intel i7 machine (2.5 GHz, 16 GB RAM, 200 GB disk), so all selected tools must run efficiently on a single workstation.

1. Industry Trends & Analysis Goals

Banking Trends: We will analyze customer transaction behavior, fraud patterns, and service usage. Key measurable trends include spending/income patterns over time, frequent transaction categories, and anomalies such as unusual withdrawals or suspicious clusters of transactions. For example, the open “MoneyData” set (6,500 retail transactions from 2015–2021) allows analysis of spending vs. income and fraud detection. Insights might include days-of-week spend peaks, average balance trends, or detection of large outliers (potential fraud). We will also track error cases like missing descriptions or abnormal amounts, and propose rectifications (e.g. flagging suspicious transactions, imputing missing fields, correcting date/time formats). Fraud patterns (e.g. rare high-value transactions or unusual merchant codes) will be studied via classification and anomaly detection models
